## Codebase Patterns
- `MapperContext` and `nextSequence` are exported from `deep-factor-agent` (re-exported from `log-mappers/types.ts`)
- `appendUnifiedSession()` in session-logger.ts is the canonical write API for unified log entries
- `getSessionId()` in session-logger.ts returns the current session UUID (creates one if needed)
- `appendSession()` is legacy, no longer called from any TUI code — function definition remains in session-logger.ts for US-012 removal

- `buildThreadFromUnifiedSession()` works directly with `UnifiedLogEntry` — no legacy conversion needed
- `SessionEntry` is internal to session-logger.ts, used only by `_convertLegacyEntry()` for reading pre-migration files
- UnifiedLogEntry types are defined in `packages/deep-factor-agent/src/unified-log.ts`
- Tests mock `deep-factor-agent` and `../src/session-logger.js` using `vi.mock()` with `vi.hoisted()` for mock references
- Pre-commit hook runs eslint with React Compiler — any `options.*` used inside `useCallback` must be in its dependency array
